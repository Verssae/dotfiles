# Vim + LazyVim 튜토리얼

> 이 파일 자체가 연습장이다. `nvim ~/.dotfiles/docs/vim-tutorial.md`로 열고,
> 직접 커서를 움직이고 지우고 되돌리면서 읽을 것. 망쳐도 됨 — `u`로 되돌리면 된다.

**가장 중요한 키 하나**: `Space`(리더 키)를 누르고 잠시 기다리면
가능한 모든 명령 메뉴(which-key)가 뜬다. 뭘 눌러야 할지 모르면 일단 `Space`.

---

## 0. 생존 키트

| 키 | 동작 |
| ---- | ------ |
| `Esc` | 모든 것을 취소하고 일반 모드로 (검색 하이라이트도 지움) |
| `u` | 실행취소 / `Ctrl-r` 다시실행 |
| `:w` | 저장 / `:q` 닫기 / `Space q q` 전부 종료 |
| `i` | 커서 앞에서 입력 시작 → `Esc`로 빠져나옴 |

vim은 **일반 모드**(이동/편집 명령)가 기본이고, `i` 등으로 **입력 모드**에
들어갔다가 `Esc`로 돌아오는 리듬이 전부다.

---

## 1. 이동 — 상대 행번호를 쓰는 이유

기본 이동: `h` ← `j` ↓ `k` ↑ `l` → (화살표도 되지만 홈로우가 빠름)

- `w` 다음 단어 / `b` 이전 단어 / `e` 단어 끝
- `0` 줄 맨 앞 / `^` 첫 글자 / `$` 줄 끝
- `gg` 파일 맨 위 / `G` 맨 아래 / `{숫자}G` 해당 줄로
- `Ctrl-d` / `Ctrl-u` 반 페이지 스크롤
- `{` / `}` 문단 단위 이동

**상대 행번호 활용** (왼쪽 숫자가 자꾸 바뀌는 그것):
가고 싶은 줄의 왼쪽 숫자를 읽고 그대로 `숫자j`(아래) 또는 `숫자k`(위).
줄 수를 셀 필요가 없다. 지금 이 줄에서 위로 5줄 가보기: `5k`

**한 줄 안에서 점프**: `f{글자}` — 그 글자로 바로 이동. `fx` 하면 다음 x로.
`;`로 반복, `,`로 반대 방향.

**화면 어디든 점프 (flash 플러그인)**: `s` + 두 글자 입력하면
화면의 해당 위치들에 라벨이 뜨고, 라벨 키를 누르면 그리로 점프.
이거 하나면 사실상 마우스가 필요 없다. 지금 `s튜토` 해보기.

연습: 아래 줄에서 `f(`로 괄호까지 간 다음 `;`로 다음 괄호로.

```text
foo(1), bar(2), baz(3)
```

## 2. 편집 — 동사 + 목적어 문법

vim 편집 명령은 `동사 + 대상` 조합이다. 이 문법만 이해하면 암기가 필요 없다.

동사: `d`(삭제) `c`(삭제 후 입력모드=바꾸기) `y`(복사) `v`(선택)

대상(텍스트 오브젝트): `w`(단어) `iw`(단어 안쪽) `i"`(따옴표 안) `i(`(괄호 안)
`ip`(문단) `it`(태그 안) — `i`는 inner, `a`는 around(구분자 포함)

조합 예시 — 아래 연습 줄에서 직접:

- `ciw` — 커서가 놓인 단어를 통째로 바꾸기 (제일 많이 씀)
- `ci"` — 따옴표 안 내용 바꾸기 (커서가 따옴표 밖에 있어도 됨!)
- `di(` — 괄호 안 삭제
- `yiw` — 단어 복사 → 다른 단어 위에서 `viwp` 하면 교체
- `dd` 줄 삭제 / `yy` 줄 복사 / `p` 붙여넣기
- `o` 아래에 새 줄 만들며 입력 / `O` 위에
- `.` — **직전 편집 반복** (ciw로 단어 하나 고치고, 다음 단어에서 `.` 누르면 같은 편집)

연습장 (마음껏 부수기):

```python
name = "hello world"
result = calculate(alpha, beta, gamma)
items = ["one", "two", "three"]
```

비주얼 모드: `v`(글자) `V`(줄) `Ctrl-v`(블록) 로 선택 후 `d`/`y`/`c`.
선택 확장은 그냥 이동키. 참고로 `Shift-v` 후 `j j`로 여러 줄 선택해 `>` 하면 들여쓰기.

## 3. 검색과 치환

- `/단어` 후 Enter — 검색. `n` 다음 / `N` 이전. `Esc`로 하이라이트 제거
- `*` — 커서가 놓인 단어를 즉시 검색
- `:%s/old/new/g` — 파일 전체 치환 (`/gc`면 하나씩 확인)
- 비주얼로 선택 후 `:s/old/new/g` — 선택 영역만 치환

## 4. LazyVim — 파일 찾기와 프로젝트 탐색

에이전트 옆 pane에서 코드 확인할 때 제일 많이 쓸 것들:

| 키 | 동작 |
| ---- | ------ |
| `Space Space` | **파일 이름으로 찾기** (fzf처럼 타이핑) |
| `Space /` | **프로젝트 전체 텍스트 검색** (live grep) |
| `Space e` | 파일 트리 토글 |
| `Space ,` | 열린 버퍼(파일) 목록에서 고르기 |
| `Space f r` | 최근 연 파일 |
| `Space s k` | 키맵 검색 — **키가 기억 안 날 때 여기서 검색** |
| `Space s h` | 도움말 검색 |

파일 트리(`Space e`) 안에서: `j/k` 이동, `Enter` 열기, `a` 새 파일, `d` 삭제,
`r` 이름변경, `?` 도움말.

버퍼 전환: `Shift-h` / `Shift-l` (이전/다음 탭처럼), `Space b d` 버퍼 닫기.

창 분할: `Space |` 세로 / `Space -` 가로, 창 사이 이동은 `Ctrl-h/j/k/l`.

## 5. 코드 읽기 — LSP

코드 파일(py/ts 등)에서 커서를 심볼 위에 놓고:

| 키 | 동작 |
| ---- | ------ |
| `gd` | 정의로 이동 (`Ctrl-o`로 되돌아오기!) |
| `gr` | 참조(사용처) 목록 |
| `K` | 문서/타입 팝업 (한 번 더 누르면 팝업 안으로 들어감) |
| `Ctrl-o` / `Ctrl-i` | 점프 히스토리 뒤로/앞으로 — gd와 세트 |
| `]d` / `[d` | 다음/이전 에러·경고로 이동 |
| `Space c d` | 현재 줄 진단 메시지 자세히 |
| `Space c a` | 코드 액션 (자동 수정 제안) |
| `Space c r` | 심볼 이름 일괄 변경 (rename) |
| `Space x x` | 프로젝트 전체 에러 목록 (Trouble) |

`gd` → 읽고 → `Ctrl-o` → 다시 `gd` ... 이 왕복이 코드 리딩의 핵심 루프.

## 6. Git

- `Space g g` — **lazygit** 열기 (스테이징/커밋/로그 전부 여기서. `q`로 닫기)
- `]h` / `[h` — 변경된 부분(hunk)으로 점프
- `Space g h s` — 커서 위치 hunk만 스테이징 / `Space g h r` 되돌리기
- 왼쪽 사인 컬럼의 `▎` 색깔: 초록=추가, 파랑=수정, 빨강=삭제

---

## 7. Markdown

- `.md` 파일은 자동으로 안에서 렌더링됨 (제목/표/체크박스). 편집하려고
  커서를 올리면 그 줄만 원문으로 보임 — 고장난 게 아니라 의도된 동작
- `Space u m` — 인라인 렌더링 켜기/끄기
- `Space c p` — 브라우저에서 실시간 프리뷰 시작 / `Space c P` 종료 (live-preview.nvim)

---

## 8. 설정 관리 (유지보수)

- `:Lazy` — 플러그인 관리 화면. `S`(sync)로 업데이트, `q`로 닫기
- `:LazyExtras` — 언어 지원 등 기능 켜고 끄기 (`x`로 토글)
- `:Mason` — LSP/포매터 설치 관리
- `:LazyHealth` — 뭔가 이상할 때 진단
- 옵션 바꾸기: `~/.dotfiles/config/nvim/lua/config/options.lua`
- 키맵 추가: 같은 폴더 `keymaps.lua`, 플러그인 추가: `lua/plugins/아무이름.lua`
- `Space u` 를 눌러보면 토글 가능한 UI 옵션 전부가 메뉴로 뜬다
  (상대 행번호 끄기도 여기: `Space u L`)
